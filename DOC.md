# TODO List 项目说明文档

## 1. 技术选型

- **编程语言**：例如 JavaScript (Node.js)，理由：生态成熟、快速开发。
- **框架/库**：例如 Nextjs/React，理由：组件化方便，利于扩展。
- **数据库/存储**：例如 Dexie，理由：轻量、浏览器本地存储、适合 demo 演示。
- 替代方案对比：例如为什么不用 MongoDB / 本地文件。用别的数据库都要安装数据库服务端，无法在本地演示。要使用后端数据库还需要实现一个用户管理模块，由于不是核心功能，所以没有必要实现。

## 3. 需求细节与决策

- 描述是否必填？如何处理空输入？ 描述非必填，空输入不做处理。
- 已完成的任务在 UI 或 CLI 中如何显示？ 已完成的任务在 UI 中使用删除线表示。
- 任务排序逻辑（默认按创建时间，用户可选按优先级）。 默认按创建时间排序，用户可通过优先级排序和截止时间排序。
- 如果涉及扩展功能（例如同步/提醒），简述设计思路。 实现了搜索待办事项的功能，由于 Dexie 并不支持模糊搜索的功能，所以使用了一个简单的正则匹配来实现搜索功能。

## 4. AI 使用说明

- 是否使用 AI 工具？使用了 Trae IDE，理由：可以快速生成代码片段
- 使用 AI 的环节：
  - 代码片段生成
  - 文档初稿编写
- AI 输出如何修改：选择合适的提示词，让 AI 生成的代码片段符合我的大体需求，再进行适当修改和封装。不要一次性生成所有代码，而是将一个大需求拆分成多个小需求，逐个生成代码片段，再进行合并。

## 5. 运行与测试方式

- 本地运行方式（安装依赖、启动命令）。
- 已测试过的环境（例如 Node.js v20，windows）。
- 已知问题与不足。没有后端数据库，所以无法实现多设备协作，而且前端数据删除后无法恢复。

## 6. 总结与反思

- 如果有更多时间，你会如何改进？
  - 增加后端数据库，实现多设备协作。
  - 增加用户管理模块，实现用户登录和注册。
  - 增加提醒/通知功能。
  - 增加批量操作功能。
- 你觉得这个实现的最大亮点是什么？
  - 实现了一个简单的 TODO List 应用，使用了 Nextjs/React 框架，Dexie 数据库，实现了基本的增删改查功能。
  - 实现了搜索待办事项的功能，使用了正则匹配来实现搜索功能。
  - 实现了任务排序的功能，用户可通过优先级排序和截止时间排序。
  - 在不依赖后端数据库的下，保证了功能的基本实现
